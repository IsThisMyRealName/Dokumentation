=== Projektidee IsThisMyRealName, 2016-04-05 ===

Ich plane eine Unterstützung für das "Mörderspiel".
( http://www.spielewiki.org/wiki/M%C3%B6rderspiel )

Was die Webanwendung können soll:
- User sollen ihre Namen (und ein Bild von sich) eingeben können.
- Jeder User bekommt den Namen eines anderen Users (dies ist sein Ziel).
- Erwischt ein User sein Ziel bekommt er das Ziel seines Ziels.
- Erwischt ein User sein Ziel bekommt er Punkte.
- User können mitteilen, wenn sie ihr Ziel erwischt haben.
- Die Punkte werden gespeichert.

Welche Elemente soll der User angezeigt bekommen?
- Registrierung mit Bild, Name, Passwort, Spielpasswort
- Namenseingabe
- Bildupload
- Aliasbildupload
- Passworteingabe
- vom Admin festgelegtes Passwort, um Fremdbeteiligung zu vermeiden
- Nachrichten
- Einloggen mit Name und Passwort
- Zielname und Bild
- Button um anzugeben, dass man das Ziel erwischt hat
- (Button um anzugeben, dass man selbst erwischt wurde)
- Logoutbutton
- Punkteanzeige


=== Mögliche Interaktive Elemente ===
-ein rein computerbasiertes Spiel aus dem Mörderspiel zu machen ist der Grundidee widerläufig.
-daher Fokus auf digitale Unterstützung

-Herrausfordern von Zielen für Bonuspunkte => ankündigen Wann/Wo man es erwischen will
-Möglichkeit des Erwischens von Zielen auch über geschickte Dateien (müsste man wohl einen Trojaner für nutzen)
-Punkte aufwenden um sich für bestimmte Zeit zu schützen (wird teurer, wenn auf einen ein Kopfgeld ausgesetzt ist, evtl. unmöglich, wenn ein Kopfgeld auf einen ausgesetzt ist)
-Tauschen von Zielen
-Ziele "versteigern" (Wenn jemand den auftrag annim
-"Kopfgelder" aussetzen (Punkte dafür, dass man das Ziel in einem bestimmten Zeitraum erwischt, evtl. Punktverlust, wenn man den Auftrag nicht rechtzeitig erledigt, oder nur eine begrenzte Anzahl an Aufträgen pro Zeit um zu verhindern, dass man einfach jedes Kopfgeld annimmt)
-GPS-Daten vom Handy, wenn man es in eine App integriert
-GPS-Daten von allen einsehbar machen

=== Feedback Alpers, 2016-04-29 ===
===================================

Leider fehlt in Ihrer Beschreibung immer noch die eigentliche Interaktion. Nochmal: Es ist Ihre Aufgabe, eine interaktive Anwendung zu erstellen, keine DB-Anwendung bzw. keinen Instant Messenger. Wenn Ihre Anwendung aber keine Programmlogik umsetzt, bzw. es keine Möglichkeit gibt, das Spiel so zu entwickeln, dass der Computer als virtueller Gegen-/Mitspieler fungiert, dann ist es keine interaktive Anwendung.

Sie müssen sich also als nächstes überlegen, wie Sie eine webbasierte Computerspiel-Version des Mörderspiels konzipieren können. Eine "digitale Unterstützung" ist keine ausreichende Grundlage für das Projekt. Das war der Grund, warum ich Ihnen schrieb, dass Sie sich interaktive Elemente überlegen sollen.
